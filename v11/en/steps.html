Using a Java User-Defined Node which reads a User-Defined Policy

IBM Knowledge Center
Learn how to deploy user-defined nodes using a provided example which reads values from a user-defined policy (requires v11.0.0.5 or higher).

Import projects

When you click Import, four projects will be created in your workspace: An application named ExampleAppUsingUDN, a policy project named ExampleUserDefinedPolicyProject, a User-Defined Node project named PolicyQuery and a Java project named PolicyQueryImpl.

The User-Defined Node project PolicyQuery provides the files describing the toolkit part of the user-defined node. Its files provide the icons for the node and describe the names of its properties and terminals. The Java project PolicyQueryImpl contains code for the implementation of the runtime of the node. The code provides getters and setters for the node properties and the main evaluate method is found inside PolicyQueryNode.java. The other two projects provide an example message flow (in ExampleUserDefinedPolicyProject) and policy(in ExampleUserDefinedPolicyProject) which can be used to test the user-defined node implementation.

Once imported using the link on the right, the application named ExampleAppUsingUDN is ready to be deployed and tested. Before doing so, the instructions on the Prepare tab describe how to set up the runtime with the code for the user-defined node.

Create an integration node owned server, configure it with the User-defined node and then deploy a flow to test it

Prepare the scenario by executing the following steps which assume a Windows platform:

  1. Create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  2. Inside PolicyQueryImpl you will find the built node as a plugin archive named PolicyQuery.par. This file needs to be made available to the ACE runtime. Copy the file to a directory named C:\temp\ExampleLilPath. Next, open the runtime integration node's configuration file (node.conf.yaml) which you will find in the ACE workpath. By default this will be at location C:\ProgramData\IBM\MQSI\components\TESTNODE.Edit the node.conf.yaml by uncommenting the line shown below, and specifying the directory where you just copied your par file:
    lilPath: 'C:\temp\ExampleLilPath' # A list of paths from where User-defined node LIL/JAR files are loaded. (multiple directories are separated by platform path separator)
    Restart the integration node.
  3. Drag and drop deploy the provided BAR file named AppUsingUDN.bar which you will find inside the ExampleAppUsingUDN application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open ExampleFlowUsingUDN.msgflow
    • You will see red crosses showing problems in the flow because at this stage your Toolkit won't know anything about the user-defined node.
    • Right click the PolicyQuery User-defined node project, and select the option Start simulation which will cause the Toolkit to load the user-defined node plugin. In the message flow, the instance of the PolicyQuery user-defined node will now be showing with its correct icon and the red crosses will have been resolved.
    • At the top of the message flow, click the Flow Exerciser icon to deploy and start testing the flow (when challenged you do not need to redeploy the flow again).
    • Once the flow has deployed, click the Send Message icon .
    • One saved input message named InputMessage is provided. Select it and click Send and your message is sent to the HTTPInput node.
    • InputMessage contains this data:
      This can be anything
    • Click on the reply message and you will see that it contains this data:
      <?xml version="1.0" encoding="UTF-8"?>
      <ResponseMessage>
        <UserDefinedPolicyProject>ExampleUserDefinedPolicyProject</UserDefinedPolicyProject>
        <UserDefinedPolicy>ExampleUserDefinedPolicy</UserDefinedPolicy>
        <UserDefinedPolicyProperty>Property2</UserDefinedPolicyProperty>
        <UserDefinedPolicyPropertyValue>ValueOfTwo</UserDefinedPolicyPropertyValue>
      </ResponseMessage>
    • After you close the dialog, the path taken through the message flow is highlighted.

This tutorial has shown how to deploy user-defined nodes using a provided example which reads values from a user-defined policy.
